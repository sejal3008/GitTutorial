Git Branching

Create a New Branch

When working on a Git repository, branching allows you to create a separate line of development, isolating your changes from the main codebase. To create a new branch:

git branch <branch_name>

This command creates a new branch with the specified name but does not switch to it.

Navigate Between Branches

Once you have multiple branches, you can switch between them using:

git checkout <branch_name>

This command switches to the specified branch.

Merge Branches

After making changes in a feature branch, you may want to incorporate those changes into the main branch. To merge branches:

git checkout main
git merge <feature_branch>

This switches to the main branch and then merges the changes from the specified feature branch into it.

Command Details

git branch <branch_name>

This command creates a new branch with the given name.

git checkout <branch_name>

This command switches to the specified branch.

git merge <feature_branch>

This command merges changes from the specified feature branch into the current branch.

Example Workflow:

  1. Create a new branch: git branch feature1
  2. Switch to the new branch: git checkout feature1
  3. Make changes and commit them.
  4. Switch back to the main branch: git checkout main
  5. Merge changes from feature branch: git merge feature1

By following these steps, you can effectively manage your Git branches and collaborate on projects with ease.
