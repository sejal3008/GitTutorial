Git Tutorial

SETUP

Configuring user information used across all local repositories

  • git config --global user.name "[firstname lastname]"
    Set a name that is identifiable for credit when reviewing version history.
  • git config --global user.email "[valid-email]"
    Set an email address that will be associated with each history marker.
  • git config --global color.ui auto
    Set automatic command line coloring for Git for easy reviewing.

Git Init Commands

The git init command is used to initialize a new Git repository in the current directory or an existing directory. Here's a list of common git init commands and their descriptions:

  • Initialize a new Git repository:
    git init
    This command initializes a new Git repository in the current directory, creating a new .git subdirectory that contains all of the necessary repository files.
  • Initialize a new Git repository with a specific directory name:
    git init <directory>
    This command initializes a new Git repository in the specified directory.
  • Initialize a bare Git repository:
    git init --bare
    This command initializes a new bare Git repository, which is typically used as a centralized repository where developers can push their changes.
  • Initialize a new Git repository with an initial commit:
    git init && git commit -m "Initial commit"
    This command initializes a new Git repository and makes an initial commit with the message "Initial commit".
  • Initialize a new Git repository with a specified initial commit message:
    git init && git commit -m "Your initial commit message"
    This command initializes a new Git repository and makes an initial commit with the specified message.
  • Initialize a Git repository in a subdirectory:
    cd <subdirectory> 
     git init
    Navigate to the desired subdirectory and then initialize a new Git repository within it.

STAGE & SNAPSHOT

Working with snapshots and the Git staging area

  • git status
    Show modified files in working directory, staged for your next commit
  • git add [file]
    Add a file as it looks now to your next commit (stage)
  • git reset [file]
    Unstage a file while retaining the changes in working directory
  • git diff
    Diff of what is changed but not staged
  • git diff --staged
    Diff of what is staged but not yet committed
  • git commit -m "[descriptive message]"
    Commit your staged content as a new commit snapshot

Generating a SSH Key

After performing all changes, you need to push all code On the Github. To do that, you need to generate an SSH key.

  1. If you already have an SSH key:
    1. Go to Github repository -> Your Profile -> Settings -> SSH and GPG key -> add new SSH keys.
    2. Give any name to it and click on Add SSH key.
  2. If you don't have an SSH key:
    1. SSH and GPG key.
    2. Click on "connecting to Github using SSH key".
    3. Generate a new SSH key, copy the URL ssh-keygen -t ed25519 -C "your_email@example.com".
    4. Open the git bash, select any folder location, paste the URL with your email address, which is connecting to GitHub.
    5. It asks for the location, set it as default, just enter.
    6. It asks for a passphrase, set it as default, just enter.
    7. Now go to the path where your SSH key is generated.
    8. Open your id_rsa.pub (name should be different) in notepad and copy the SSH key.
    9. Now come back to Github -> SSH and GPG key -> add new SSH key, paste the URL in the description.
    10. Click on Add SSH key.
  3. Now Go to the repository -> local -> ssh -> copy the URL and add.
  4. Before push, you have to set the remote repo:
    • Run the following command: git remote add origin [url] (paste the URL here).
    • Now you can easily push the changes on GitHub by running: git push origin main (this will push changes from local to your Git repository).

SHARE & UPDATE

Retrieving updates from another repository and updating local repos:

  • git remote add [alias] [url]
    Add a git URL as an alias
  • git fetch [alias]
    Fetch down all the branches from that Git remote
  • git merge [alias]/[branch]
    Merge a remote branch into your current branch to bring it up to date
  • git push [alias] [branch]
    Transmit local branch commits to the remote repository branch
  • git pull
    Fetch and merge any commits from the tracking remote branch

INSPECT & COMPARE

Examining logs, diffs, and object information:

  • git log
    Show the commit history for the currently active branch
  • git log branchB..branchA
    Show the commits on branchA that are not on branchB
  • git log --follow [file]
    Show the commits that changed file, even across renames
  • git diff branchB...branchA
    Show the diff of what is in branchA that is not in branchB
  • git show [SHA]
    Show any object in Git in human-readable format
